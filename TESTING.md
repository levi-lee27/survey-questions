## 测试步骤

### 准备工作
```bash
cd survey-web
python -m http.server 8080
# 或使用: npx serve .
```

访问: http://localhost:8080/generate.html

### 第一步: 创建问卷
1. 在「问卷标题」输入框填写: `测试问卷`
2. (可选)在「管理员密码」输入: `test123`
3. 点击「生成问卷链接和二维码」按钮
4. 验证:
   - ✅ 显示问卷链接和后台链接
   - ✅ 二维码正常显示
   - ✅ 右侧列表出现新问卷

### 第二步: 填写问卷
1. 复制问卷链接,在新标签页打开
   ```
   http://localhost:8080/index.html?surveyId=survey_xxx&title=测试问卷
   ```
2. 验证页面显示:
   - ✅ 问卷标题正确显示 "测试问卷"
   - ✅ 三个评分问题正常显示
   - ✅ 提交按钮初始为禁用状态
3. 选择所有三个问题的选项
4. 验证:
   - ✅ 提交按钮变为可用
5. (可选)填写建议内容
6. 点击「提交评价」按钮
7. 验证:
   - ✅ 显示成功弹窗,包含各题分数和总分
   - ✅ 弹窗 3 秒后自动关闭
8. 再次填写并提交(多次,如 5 次)

### 第三步: 查看统计
1. 访问后台链接:
   ```
   http://localhost:8080/admin.html?surveyId=survey_xxx
   ```
   - 如果设置了密码: `http://localhost:8080/admin.html?surveyId=survey_xxx&token=test123`
2. 如果设置了密码:
   - ✅ 显示密码输入框
   - 输入密码 `test123`,点击「解锁统计」
   - ✅ 验证通过,进入统计页面
3. 验证统计页面:
   - ✅ 顶部显示: 总提交数、平均分、最高分
   - ✅ 各题平均分显示正确(带进度条)
   - ✅ 提交记录列表完整显示
   - ✅ 每条记录包含时间、各题分数、总分、建议
4. 测试筛选和排序:
   - ✅ 搜索框输入建议关键词,列表实时过滤
   - ✅ 下拉菜单切换排序(时间/分数,升序/降序)
5. 测试 CSV 导出:
   - ✅ 点击「导出 CSV」按钮
   - ✅ 文件自动下载
   - ✅ 用 Excel 打开验证数据完整
6. 测试分页(如有足够数据):
   - ✅ 显示分页控件
   - ✅ 页码导航正常
7. 点击右上角「← 返回问卷」链接:
   - ✅ 正确跳转到问卷页面

### 第四步: 多问卷隔离
1. 回到 generate.html,创建第二个问卷(不同标题,不同密码)
2. 填写并提交数据
3. 验证:
   - ✅ 每个问卷的数据完全隔离
   - ✅ 访问各自的 admin.html 只能看到对应数据

## 自动化验证

运行集成测试脚本:
```bash
python integration_test.py
```

预期输出:
- [OK] 创建问卷成功
- [OK] 提交成功,显示分数
- [STATS] 统计结果正确
- [SUCCESS] 所有测试通过

## 已知限制

- 数据存储在浏览器 localStorage,清除浏览器数据会丢失
- 不同浏览器/设备间数据不共享
- 适合内部轻量级使用,生产环境建议后端持久化
